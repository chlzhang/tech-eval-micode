# 工作流程参考

## 三阶段并行流水线详细说明

### Phase 1：并行读取与检索

在同一条消息中同时启动两个子代理（使用 actor 工具并行调用）：

#### Agent-A「素材读取与结构化提取」

**输入：**
- `input/transcript.md` - 会议转写文本
- `input/counterpart/` - 对方技术文档目录
- `input/benchmark/` - 我方参考基准文档目录

**输出：**
- `output/.tmp_materials.md` - 结构化素材包

**处理流程：**
1. 使用 Glob 工具发现所有输入文件
2. 逐个 Read 文件内容
3. 提取结构化信息：
   - 对方身份：单位名称、主营业务、核心人员
   - 我方身份：参与人员、项目背景
   - 交流形式与时间
   - 对方技术主张：逐条列出（原理、指标、成熟度、案例），每条标注来源
   - 对方含糊/回避的问题清单
   - 我方现场观察（如有）
   - 我方基准文档关键数据
   - 结构化对比数据（key-value 对形式）

**关键要求：**
- 严格区分事实与判断
- 所有对方主张必须标注来源
- 遗漏信息不得脑补

#### Agent-B「行业基准检索」

**输入：**
- `input/transcript.md` - 用于提取核心技术主题和关键数值

**输出：**
- `output/.tmp_benchmarks.md` - 结构化行业基准摘要

**处理流程：**
1. 读取转写文本，提取核心技术主题
2. 提取对方主张中的关键数值（温度、时间、产能、效率、成本等）
3. 精准化搜索关键词（包含具体技术路线）
4. 执行 7 次并行搜索：
   - 标准原文搜索
   - 国际标准搜索
   - 经济数据搜索
   - 技术参数搜索
   - 合规风险搜索
   - 对方核实搜索
   - 技术路线横向对比搜索
5. 整理搜索结果为结构化格式

**输出格式：**
每条基准包含：
- 内容
- 来源 URL
- 来源类型（国标原文 > 国际标准 > 行业报告 > 学术论文 > 新闻 > 模型知识）
- 置信度（高/中/低）
- 决策含义

**关键要求：**
- 优先获取标准原文条款号
- 对方主张与标准矛盾时显式标注
- 主动搜否定性证据
- 必须搜到经济数据
- 经济数据必须标注所属技术路线
- 不同路线的经济数据不得混用

### Phase 2：报告撰写

等待 Agent-A 和 Agent-B 均完成后启动 Agent-C。

#### Agent-C「报告合成」

**输入：**
- `output/.tmp_materials.md` - Phase 1 素材包
- `output/.tmp_benchmarks.md` - Phase 1 行业基准包

**输出：**
- `output/report_draft.md` - 完整报告草稿
- `output/report_data.json` - 结构化图表数据

**处理流程：**

**第一步：撰写报告**
1. 读取两个临时文件
2. 根据版本选择报告结构（精简版 4 章 / 完整版 8 章）
3. 遵循四条铁律撰写报告

**第二步：CRITIC 对抗审查**
1. 以审查者视角重新审视报告
2. 检查项目：
   - 矛盾扫描
   - 薄弱证据识别
   - 偏见审计
   - 隐含假设检查
   - 缺失角度
3. 将审查发现写入报告末尾附注

**第三步：FACT-CHECKER 数值核验**
1. 提取所有数值声明
2. 逐条核验：
   - 是否有素材原文或搜索结果支撑
   - 计算是否正确
   - 标准条款号是否准确
3. 修正错误数值
4. 标注无法核实的数值
5. 将核验结果写入报告末尾附注

**最后：写入最终报告并清理**
1. 写入 `output/report_draft.md`
2. 提取结构化图表数据写入 `output/report_data.json`
3. 删除临时文件

### Phase 3：双格式输出

#### 3a. DOCX 输出

使用 minimax-docx 技能将 Markdown 报告转换为 Word 文档。

**输入：**
- `output/report_draft.md`

**输出：**
- `output/技术评估报告_{YYYYMMDD}_{技术主题}.docx`

#### 3b. HTML 可视化报告输出

生成单文件 HTML 报告，内嵌 CSS 和 JS。

**输入：**
- `output/report_draft.md`
- `output/report_data.json`

**输出：**
- `output/技术评估报告_{YYYYMMDD}_{技术主题}.html`

**要求：**
- 使用 Chart.js 渲染图表（通过 CDN 引入）
- CJK 专业排版
- 交互功能（折叠、斑马纹、可点击链接）
- 左侧/顶部目录锚点跳转
- 打印样式优化

## 素材不足时的降级策略

| 情况 | 处理方式 |
|------|----------|
| 仅有转写文本、无行业基准文档 | 仍可完成报告，在第四章标注"行业基准来源为模型训练知识，未经本次专项检索验证" |
| 素材信息极度稀少 | 在执行摘要中说明"因素材有限，以下评估置信度较低"，降低结论强度，加粗待核实清单 |
| 无我方现场观察 | 跳过第三章，在报告开头注明"本次报告不含我方实地观察" |

## 错误处理

- **文件读取失败**：记录错误并继续处理其他文件，在报告中标注缺失内容
- **搜索失败**：使用模型知识作为备选，但标注置信度为"低"
- **格式转换失败**：提供原始 Markdown 文件作为备选
